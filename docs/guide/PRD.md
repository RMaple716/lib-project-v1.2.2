### **1. 项目概述**

*   **产品愿景**：主要为高校学生和教师提供一个安全、便捷、专属的图书查询和节约平台，降低部分寻书成本。
*   **问题陈述**：
    *   学生找不到书时只能去其他图书馆寻找，花费的时间成本较多。
    *   学生在找到馆藏已全部借出的图书时，只能等待馆藏更新时再来借阅，等待时间较长。
    *   学生在借阅时，常常因为一些非主观原因忘记及时归还。
*   **项目范围**：
    *   **本项目将包括**：基于学号的实名认证、图书信息发布、搜索与浏览、新书建购
    *   **本项目不包括**：书籍购买、在线支付、物流配送、文献检索、书籍质量评级系统。

### **2. 用户角色与画像**

| 角色名称 | 核心特征与需求 |
| :--- | :--- |
| **学生** | - **特征**：对非学术资料的书籍需求较高，对市面上的热门书籍较为感兴趣<br>- **需求**：在不知道读什么书时可以从系统中快速找到自己感兴趣的书籍 |
| **教师** | - **特征**：对学术资料需求较高，借阅书籍量较多<br>- **需求**：方便地找到所需学术资料，对于未找到的学术资料可以在较短时间的等待之后获得 |
| **社会人员读者** | - **特征**：对访问学校图书馆较为感兴趣<br>- **需求**：借阅部分书籍，对于未找到的书籍可以通过去其他地方寻找获得，对预约功能要求不高 |
| **管理员** | - **特征**：对图书馆的书籍信息有全面了解，对图书馆的书籍借阅情况有全面掌握<br>- **需求**：对图书馆的书籍信息进行管理，对图书馆的书籍借阅情况进行管理 |

### **3. 功能需求列表（用户故事格式）**

*   **用户认证与安全管理**
    1.  作为一名读者，我希望能够用我的学号和教务系统密码进行验证，以确保平台用户均为本校真实学生。
    2.  作为一名已认证用户，我希望能够登录和登出我的账户，以保护我的隐私。
    3.  作为一名已认证用户，我希望能够修改我的密码，以保护我的账户安全。
    4.  作为一名社会人员读者，我希望可以通过临时建立账号来访问平台，以便快速浏览图书信息。

*   **图书浏览与搜索**
    3.  作为一名学生读者，我希望在首页按标签、图书名称、作者来浏览图书，以便发现我可能需要的图书。
    4.  作为一名教师读者，我希望通过输入ISBN号或书名直接搜索图书，以便快速定位目标。

*   **图书预约**
    5.  作为一名读者，在找到心仪的暂时无馆藏可借阅复本书籍后，我希望能通过平台内置的预约功能进行图书预约，以便在有馆藏复本时及时去借阅。
    6.  作为一名读者，我希望可以和图书馆相关人员联系以合理要求图书馆员购进一些需要的书籍以供借阅。

### **4. 非功能性需求**

*   **性能**：主要页面的加载时间应在3秒以内。
*   **安全性**：用户密码必须加密存储；所有用户必须经过学籍验证；限制外部人员的部分访问。
*   **兼容性**：平台应为响应式网站，能在主流浏览器（Chrome, Safari）及手机浏览器上正常显示和操作。
*   **易用性**：界面设计简洁明了，符合学生群体的使用习惯，不会存在繁琐的操作流程。

### **5. 部分功能详述**

**功能名称：图书查询流程**

*   **描述**：已验证身份的读者可以在系统上按条件搜寻想要找的图书。
*   **前置条件**：用户已登录。
*   **交互流程与逻辑**：
    1.  用户点击首页就有的的搜索框。
    2.  用户需填写/操作以下字段之一并在搜座框左侧选择对应字段：
        *   **ISBN号（选泽后填入）**：输入后系统尝试从本地数据内获取书名、作者、出版社等信息。
        *   **书名（选择后填入）**：书名应在30字以内
        *   **作者（选择后填入）**：作者名应在15字以内
    3.  点击“搜索”按钮，系统校验输入信息后，搜索到的书籍信息展示在系统上，包括书名，作者，馆藏数量，借出复本，可借复本等。


---

**功能名称：图书管理员对图书的添加**

*   **描述**：图书管理员可以在图书录入模块单本图书录入也可以批量录入。
*   **前置条件**：图书管理员已登录（未登录也可浏览，但联系卖家需要登录）。
*   **交互流程与逻辑**：
    1. 图书管理员点击页面中的“导入图书功能” 
    2. 图书管理员选择导入方式，单本导入或批量导入
    3. 选择单本导入后输入图书信息、作者、标签等书籍基本信息，输入后点击”确认“按钮，系统自动将图书信息录入到数据库中
    4. 选择多本导入之后点击“选择”按钮，从本地文件内选择整理为特定格式的图书信息的excel文件，点击“导入”按钮，系统自动读取文件内容并导入图书信息

### **6. 初步成功指标**

*   在发布之后帮助200名读者找到需要的书籍
*   平台上线后延期未归还的书籍记录不超过10条

### **7. 待定事项与风险**

*   **待定事项**：
    *   是否允许用户对搜索进行评价？
    *   是否采用深度学习的方式训练推荐功能模块？
    *   如果自动获取图书信息失败，是否有备用方案？
    *   是否允许用户上传图书信息？

*   **风险**：
    *   技术风险：可能无法找到稳定的图书信息API接口。
    *   运营风险：初期可能因用户量少，无法形成有效的图书推荐（冷启动问题）。
    *   安全风险：该系统规模较小，可能无法抵抗某些恶意攻击。
